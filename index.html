FAQs

  • > 'I've lost/forgotten my password'

    You can reset your password your password by clicking the 'Forgotten password' link on the login page, or by clicking here.
  • > 'My password is no longer working'

    Please try resetting your password by following the above steps. If you are still unable to log in, please ensure you are using Luminance in Chrome or Edge (Firefox support coming soon!). For security reasons some of our clients request we limit the IP addresses from which employees can login, please ensure you are logged into the correct network or VPN. Contact your system administrator if you are unsure whether you are logged into the correct network, or Contact Us if you still require our assistance.
  • > 'Incorrect number of documents'

    If you have recently deleted or uploaded a large number of documents, please be aware it may take a few minutes (up to an hour) for the correct number of documents to display.
  • > 'Slowness/Buffering'

    Please check your internet connection and try logging out and logging back into your account. If the slowness/buffering persists please Contact Us.
  • > 'My Upload is stuck!'

    We do not recommend making large uploads (>5GB) directly through your browser. Please try compressing the files into a .zip, uploading the files in smaller batches or Contact Us, and one of our team will discuss alternatives for the secure upload of your documents.

Internal API Guide (Sample)

What is LumPy?

LumPy is a Python based RESTful API which will allow internal teams to carryout basic operations. RESTful APIs work through the construction of URLs to reach specific end points. To construct the beginning of any URL, first generate an access token here, then specify a path.

Examples of endpoints include: '../documents' '../users' and '../document_events'.

In the same way that URLs are used by your browser, LumPy uses HTTP methods, such as GET, POST, PATCH, and DELETE. Here are some examples of these methods in action:

'I want to find all documents that are upload_state: pending'
documents = .. GET ..<room_id>/documents
for doc in documents if upload_state=='pending'
print documents.id

'I want to delete a user/document'
DELETE ../users/<user_id>
DELETE ../documents/id=<doc_id>

'I want to find all deletions made by a specific user'
GET ../document_events/user_id=<user_id>/event='delete'

'I want to change user permissions'
PATCH ../users/ id=<user_id>/access='admin'

'I want to create a new user'
POST -d {"username": "<email>", "access": "nonadmin"}' ../rooms/id=<room_id>/users
